#### Intro to HTML

##### Welcome to the web!

The "web" is a network of connected computers. Servers serve out website data. Clients are our devices we use to browse the internet.

Websites typically utilse three languages: HTML for marking up content, CSS for styling, and JavaScript for interactivity.

##### HTML basics

The whole webpage is contained within <html> tags. Within this, there are the <head> and <body> tags.

Within the <head> tag, we have data that helps the browser render the page, but doesn't include anything the user actually sees within the page.

<meta charset="UTF-8"> is commonly included here to indicate the character set the page will render. utf-8 is commonly used as it includes the most common English characters.

You should include You may contain external CSS and JavaScript links here, or metadata, such as page title (<title>), website icon, or embedded thumbnail image.

  • <h1> - headline tag

  • <br> - line break

##### CSS: Selecting by id

Covers assigning id's so that we can select id'd elements directly in CSS.

First, assign id in element tag like so: <p id="tag-id">

Then in the CSS <style> section of <head> we can select this id like so: #tag-id {}, adding our styles between the curly brackets.

id's are case sensitive, must not have spaces, and must be unique.

##### CSS: Selecting by class

Covers assigning classes so that we can select elements directly in CSS with class. We do this when we want to style multiple elements.

First, assign class in element tag like so: <p class="tag-id">

Then in the CSS <style> section of <head> we can select this class like so: .tag-id {}, adding our styles between the curly brackets.

Classes are also case sensitive and must not have spaces.

#### More HTML tags

##### HTML links

To link to other HTML pages, we use hyperlinks, or just links. To tag we use to do this is the anchor tag, or <a>.

We refer to the target link with a hyper reference ie. href="link.html" inside the tag.

We can also add target="_blank" to ensure the link opens in a new tab. This should be used for links targetting an outside domain.

The link text will be the text contained between the opening and closing <a> tags.

##### HTML internal links

Links can be used to navigate within a page, ie a table of contents.

We can do this using id's. Add an id to the leading element for the section you want to link to, and then when creating the link itself in your contents, define the href as that id, preceeded by a hash (#) symbol.

Define the section id:

<h1 id="project-1">
    Project 1
<h1>

Internal link to section:

<a href="#project-1">
    Internal link to Project 1
<h1>

External link to section:

<a target="_blank" href="https://website.com/index.html#project-1">
    External link to Project 1
<h1>

This is why we ensure id's are unique, as id's are typically used for functionality such as this and JavaScript.

Classes are reserved for styling.

##### HTML comments

Sometimes we want to write into our code for the benefit of ourselves or future editors. We do this with comments. These will not render on the front-end, but will be visible in the code if viewed.

To do this we use <!-- followed by our comment, and ending with -->

This is also useful for making unused content or code invisible to the user, without erasing it entirely.

#### CSS layout

##### CSS grouping elements

Now, using CSS to change the layout of the page. This is the part that's a total ballache. We can use existing tags to configure our layout, but sometimes we want to arrange groups or parts of these elements.

To do this, we must introduce two new HTML tags which we call grouping elements.

First, the <span> tag. This is used for grouping selections of text within other tags.
Second, the <div> tag. This is used for grouping selections of multiple tag groups.

By assigning a class name to either of these tags, we can then selectively apply CSS styling by referring to the class.

Now. A new annoying thing to contend with. Two elements that exist in the cursed world of CSS: inline and block.

An inline element does not have a new line after it. If you have multiple of them, they'll all be on one line. Some tags that are inline elements by default are <a> and <img>.

A block element does have a line after it. For example, the <h*>, <p>, <li> tags always have a line break by default.

So, from looking at <span> and <div> we can see that <span> creates inline elements, while <div> creates block elements.

From such humble beginnings, the mighty <div> tag announces its presence, with only the slightest murmurings of its future dominance. Remember the mighty <div>.

##### CSS width, height, and overflow

When we use divs, it's often for the purpose of moving and resizing our elements as boxes. It takes a good eye to discern whether a page is layed out well, and the iron patience of a tired parent to fight with CSS to make it sit still.

First we can use the CSS element width to set an absolute width for our div element, or as a percentage of available width (as defined by surrounding elements and ultimately the device width). The text will wrap to fit the width of the box, thus also extending its height.

We can also constrain the height> of our div if we want, but by doing this we may cause an unsightly overflow, if the space is insufficient for the content.

We can control overflow rules with the CSS element overflow. By default, overflow is set to visible, which means the content will flow outside of the constraints of the div block.

There are other options for overflow, however. Such as hidden, which will make any text beyond the limits of the div invisible. However, this means anyone viewing the page can't read the rest of the content.

More ideally, we can set overflow to auto, which will allow the user to scroll within the div.

Knowing this though, remember that scroll bars are annoying and nobody likes them. So don't use them, unless you're in a very dire situation which requires you to commit such a heinous sin.

We can also define the margin per side, starting from the top and going clockwise, as so:

div{margin:10px 15px 20px 5px}

The above would set the top margin as 10px, right margin as 15px, bottom margin 20px, and left margin 5px.

div{margin:10px 15px}

Additionally, blank size definitions will default to the size of its opposing sides. In the above, the top and bottom margin will be 10px, while the left and right margins will be 15px.

div{margin:auto}

##### CSS Position

In the example piece, inline elements and block elements are displayed according to default CSS positioning strategy. We want to change this using the CSS position property.

For the first element, a landscape picture, position is set to relative. This tells the browser to display the element as it normally would, but then we will offset this. For example, we want to offset this 20px from the top, and 10px from the left. To do this we can add the following to the CSS style rules:

img{
    top: 20px;
    left:10px;
}

Now, that's cool. It renders the image over other stuff now, offset from it's relative default position. However, we can do better, by changing relative to absolute.

First, by setting the position to absolute, this changes the position of the following elements inline with the image being changed, as the absolute positioning removes it from the relative positioning rules.

If we are setting out images with absolute positioning, we now have to contend with content being drawin over other content. We might not agree with the order. Content will be rendered in order of the HTML, so we can choose to change this. However, we can also change the positioning within the CSS using the z-index property.

An element with z-index 1 will be lower than an element with z-index 3. They stack like this.

Default z-index of elements will be 0

By scrolling the page, we can see an important aspect of absolute positioning: elements are ordered relative to the top left of the page.

However, maybe you want the position to remain? Maybe you want the header to be visible regardless of where you've scrolled on page. In this case, you want it to be fixed. The position of the element will remain the same regardless of where you are on the page. It is relative to the top of the viewing window, rather than the page itself.

##### CSS floating elements

Now, we're looking at float. Float is another beast. It is used for wrapping elements in and around other elements. In the case of this example in the course, we're going to use it to wrap text around an image.

img {float:left;}

Generally when we try to float a <div> it attempts to take up all the space, so we should always assign a width.

In the case where a sidebar element as described above overlaps the footer, we can ensure that the footer element clear of the floating content on both left and right by using:

footer {clear:both;}
##### Planning your webpage

Going straight into HTML and CSS with no planning can lead to a lot of wasted time. Instead, you should figure out content, layout, and style beforehand.

Brainstorming

Brainstorming by using mind maps, jotting notes, utilising post-it notes. Collaborating with someone can help this process, weeding out dud ideas, catching blind-spots, etc. The Yes, And approach can help maintain forward momentum when brainstorming ideas. Once ideas have been generated, you can begin connecting the best ideas and scrutinising those that don't fit.

Paper prototyping

Prototyping your page on paper can be a faster way to iterate through ideas before getting caught up in the technical headaches of CSS. This can give you a version of your website to further scrutinise, and which to show to friends to test your design concept before going ahead with production.

Wireframes

A wireframe is the next level up from the paper prototype. A more precise digital sketch of your design concept to present to clients and collaborators that present the application in context.

Some popular tools for prototyping include:

  • Figma
  • Balsamic
  • Powerpoint
  • Photoshop
  • Gimp
  • InDesign
